### 一、 素材感知与采集 (Material Perception & Collection)

本模块侧重于视频与评论数据的自动化获取与持久化，将无结构的社交媒体内容转化为结构化的资产。

**（1）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">定向 URL 异步抓取逻辑</span>
- **业务逻辑**：创作者在系统控制台配置抓取队列或采集主题包。提交后，前端立刻响应“任务已建立”，后台随即触发并静默收集视频文件与海量评论数据。
- **实现原理**：为了防止 Django 主线程被长轮询请求锁死导致 HTTP 504，爬虫被分发至 `threading.Thread` 的异步守护线程。采用了 `DrissionPage` 控制 Chromium 无头浏览器，并利用全局缓存（Redis / 本地内存）进行 ETA 进度通讯。数据库操作采用了 `UnifiedPersistenceManager`，实现单例级别锁和事务解耦。

```python
# *=== 异步数据库调度 (views.py) ===*
def _run_spider_background(keywords_list, max_videos, max_comments, theme_name):
    # *向 sys.path 注入采集根目录*
    sys.path.insert(0, os.path.join(settings.BASE_DIR, 'data'))
    from crawler.spyder_unified import DouyinUnifiedPipeline
    
    # *ETA 缓存心跳与状态打点*
    def progress_callback(current, total, start_time, message=""):
        # *向全局 cache 推送 ETA 指标和阶段日志，与主线程物理隔离*
        cache.set('spider_progress', {
            'percent': int((current / max(1, total)) * 100),
            'status': message
        }, timeout=300)
    
    # *解耦启动统一爬虫管道*
    DouyinUnifiedPipeline().execute_task(
        keywords_list, max_videos, max_comments, theme_name, progress_callback
    )
```

### 二、 AI 预测实验室 (AI Prediction Lab)

结合本地化的大模型与 3060 GPU 算力，对素材包进行物理维度解构，最终将多源维度送入预测模型。

**（1）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">视觉/听觉特征张量提取</span>
- **业务逻辑**：当一个新视频进入 AI 待处理队列后，后台“数字监工”自动接管并解析出视频的物理明暗、色彩饱和与听觉节奏。
- **实现原理**：后台线程唤醒 `MultiModalFeatureFactory`。视觉方面，基于 OpenCV 抽取并利用 `torch.from_numpy().to('cuda')` 将矩阵送入 GPU 进行均值计算。为防止高分辨率视频在 3060 下**显存溢出 (OOM)**，代码实施了“隔帧提取（每 5 帧提取 1 帧）”与“早停（最大处理 500 帧）”的截断回收机制。听觉方面则使用音频引擎 `Librosa` 检测 BPM 量化规律。

```python
# *=== 视觉 GPU 计算核心 (feature_factory_v2.py) ===*
def extract_visual_on_gpu(self, video_path):
    cap = cv2.VideoCapture(video_path)
    # *内存回收机制：强制扫描最多 500 帧*
    while cap.isOpened() and total_processed_frames <= 500:
        ret, frame = cap.read()
        if not ret: break
        
        # *隔帧提取，减少总张量负荷*
        if total_processed_frames % 5 == 0:
            # *数据投递至 3060 CUDA, 快速计算均值和亮度*
            t_frame = torch.from_numpy(frame).float().to(DEVICE)
            brightness_list.append(torch.mean(t_frame).item())
            # *...省略直方图对比计算...*
        total_processed_frames += 1
    
    cap.release() # *释放句柄*
    return np.mean(brightness_list), cut_freq
```

### 三、 多维洞察可视化 (Multi-dimensional Insight Visualization)

从数字废墟中还原创作者的用户画像与宏观规律，输出高颜值 ECharts 中控台。

**（1）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">深度词云降维引擎</span>
- **业务逻辑**：面对单主题下万级别的凌乱留言，系统需直接投射其核心用户情感词汇到大屏词云。
- **实现原理**：首先屏蔽默认系统分词（如“的”、“了”、“我们”），采用 `jieba.analyse.extract_tags` 运用 TF-IDF 算法提取最核心语义特征（Top 5）。在视图层汇聚并通过 Python `Counter` 统计频段结构，转化为前台 ECharts 所需要的 JSON 拓扑节点。

**（2）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">ECharts 并发渲染处理器</span>
- **业务逻辑**：呈现包括音视觉分布（BPM、切割频率）以及 4D 爆款基因散点图的用户交互。
- **实现原理**：过滤掉无效的 null 数据（`exclude(visual_brightness__isnull=True)`）。在内存中使用 Django ORM 将数据按区段分发：如 `audio_dist['bpm']['fast'] += 1`，将 Python 的 Dictionaries 格式化打包为 `json.dumps()` 传输至前台，避免客户端产生冗长的转换开销。

```python
# *=== ECharts 结构转换器 (views.py) ===*
def chart_content(request):
    top_dna_data = []
    # *提取排名前列的赛博基因为散点图提供数据系*
    valid_videos = Video.objects.filter(theme_label=active_theme, analysis_status=2)
    top_videos = valid_videos.order_by('-digg_count')[:10]
    
    for v in top_videos:
        # *将四维参数封装为前台要求的数据节点*
        top_dna_data.append({
            'name': f"{v.nickname} - {v.video_id[-4:]}",
            'value': [
                float(v.visual_brightness or 0), 
                float(v.visual_saturation or 0), 
                int(v.audio_bpm or 0), 
                float(v.cut_frequency or 0) * 100, 
                min(int(v.digg_count or 0) / 1000.0, 100.0)
            ]
        })
    return render(request, 'charts.html', {'top_dna_json': json.dumps(top_dna_data)})
```

### 四、 智能运营策略 (Intelligent Operations Strategy)

提供最终的赛道定准并融合文本大语言模型，构成创作者的专属 AI 诊断导师。

**（1）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">模型交互与 2.0s 熔断保护</span>
- **业务逻辑**：系统基于物理指标与机器预估跑分，结合自设的角色生成诊断 Prompt，提供 200 字以内的专业“手术方案”。
- **实现原理**：构建基于百度千帆（默认）及动态 OpenAI 层级访问的 `VideoAdviser` 连接。考虑到模型响应有可能发生无限期挂起进而锁死 Django 核心预测管道，系统特别启用了 `concurrent.futures.ThreadPoolExecutor` 隔离调度。若在 2.0 秒内未能响应则强制触发“熔断异常抛出”，执行无盘回退降级策略。

```python
# *=== LLM 交互与 2.0s 超时熔断 (views.py) ===*
from concurrent.futures import ThreadPoolExecutor

# *将质量分与多维指标整合构造系统级诊断 Prompt*
advice_context = {
    **p_data, 'predicted_likes': pred_likes, 'theme_name': theme_name
}

try:
    # *设置工作线程池以保证 Django 主并发安全*
    with ThreadPoolExecutor(max_workers=1) as executor:
        future = executor.submit(VideoAdviser().generate_advice, advice_context, user_key=user_key)
        # *实施严格的 2.0 秒并发熔断 (Circuit Breaker)*
        advice = future.result(timeout=2.0)
except Exception as e:
    # *若 API 受限或响应迟缓，快速防雪崩降级*
    print(f"[Circuit Breaker] LLM Timeout: {e}")
    advice = "系统负载较高，启用降级防护：建议提升前3秒对比度，并在文案设置引导性提问。"
```

**（2）** <span style="color:#00f0ff; font-size:20px; font-weight:bold;">主题基准统计平滑器</span>
- **业务逻辑**：模型预估的爆量在部分小众长尾赛道中没有参考意义，需要采用 P25 - P75 的分位基准，实施校准。
- **实现原理**：不再盲目信任单一预测树节点的绝对反馈值，`ThemeBaselineCalculator` 计算子类赛道内的中位数边界，若样本太少（稀疏矩阵），则应用贝叶斯先验概率（Bayesian Smoothing）将大盘总体方差平滑注入局部主题，计算后反套入算法返回给用户极有参考意义的 `percentile_rank`。

# 第X章 系统架构设计

## X.1 系统逻辑架构概述
本系统（多模态短视频智能分析系统）采用经典的四层逻辑架构（4-Layer Logical Architecture）进行设计。该架构自上而下分为表现层（Presentation Layer）、业务逻辑层（Business Logic Layer）、数据访问层（Data Access Layer）与基础设施层（Infrastructure Layer）。这种分层设计有效实现了系统各模块之间的解耦，确保了系统的高内聚、低耦合特性。同时，通过合理的组件映射，能够最大化利用硬件资源（特别是RTX 3060 GPU的CUDA计算能力），保障系统在处理多模态海量数据时的高效性与可扩展性。

## X.2 四层逻辑架构详细定义与组件映射

### X.2.1 表现层 (Presentation Layer)
表现层是系统与各类终端用户（如普通系统用户、系统管理员等）进行直接交互的接口。其核心职责是接收用户交互指令、展示数据分析结果以及提供系统的操作界面。
*   **分配系统组件：**
    *   **动态网页渲染组件：** 基于 Django Templates 实现后端驱动的页面动态渲染及表单提交逻辑。
    *   **后台管理控制台：** 采用 SimpleUI 构建现代化的系统管理员管理面板，支持资产与系统权限配置。
    *   **业务数据看板：** 集成 ECharts 与 Chart.js 库，基于前端 DOM 提供多维度的数据可视化（例如：智能推荐漏斗、情感趋势曲线等）。
*   **核心功能映射：** 承载用户的“身份验证登录”、“工作流导航”、“可视化仪表盘展示”以及“分析任务下发”等交互级功能点。

### X.2.2 业务逻辑层 (Business Logic Layer)
业务逻辑层是系统的智能决策与任务调度的“大脑”，负责处理核心业务规则、核心算法调度以及多模态特征分析的复杂逻辑。本层承载了最高密度的学术与工程实现代码。
*   **分配系统组件：**
    *   **核心应用框架：** 运用 Django Framework 实现业务路由解析、请求分发及内部数据序列化。
    *   **异步任务调度器：** 采用 Django-Q 取代阻塞式的本地执行引擎，构建隔离的异步任务队列（也可平滑替换为 Celery），以防高负载的视频解析与模型推理阻塞主控线程。
    *   **智能特征提取引擎：**
        *   **视觉特征提取器：** 利用 OpenCV 处理短视频逐帧图像矩阵，抽取光流帧及视频色彩直方图等核心视觉特征。
        *   **音频特征提取器：** 引入 Librosa 处理短视频音轨，基于数字信号处理技术提取梅尔频率倒谱系数 (MFCC) 及频带能量特征。
    *   **核心机器学习组件：** 构建 XGBoost Predictor 预测容器，接收多模态量化后的特征张量融合进行决策推演与回归分析。
*   **核心功能映射：** 对应需求链路中的“多模态指标量化（Multimodal Quantification）”、“智能分析与决策支持（Intelligence & Decision Support）”及“个性化推荐（Personalization）”的算法层工作流支撑。

### X.2.3 数据访问层 (Data Access Layer)
数据访问层在业务逻辑与底层物理存储架构之间构建标准接口桥梁，负责实现域对象的数据持久化管理与交互读写操作，从根本上屏蔽底层关系型数据库的语法级细节。
*   **分配系统组件：**
    *   **对象关系映射 (ORM) 引擎：** 启用 Django 核心架构的 ORM 组件，将系统的核心 Python 对象映射至结构化数据库实体模型，保证对象持久性。
    *   **一致性处理与安全防范：** 封装高可用的安全 CRUD（创建、读取、更新、删除）抽象层（例如防护 SQL 注入），具备事务回滚级支持。
*   **核心功能映射：** 执行“多模态数字资产持久化（Asset Management）”、“模型权重版本控制持久化”以及爬虫入库日志的写入和检索操作。

### X.2.4 基础设施层 (Infrastructure Layer)
基础设施层是支撑整个逻辑体系运转的物理硬件、底层操作系统和外挂的第三方服务网关集群。系统重度依赖此层提供的大规模异构计算与带宽支持。
*   **分配系统组件：**
    *   **宿主运行环境基座：** 基于 Windows 11 专业操作系统平台提供驱动支持与系统调度池。
    *   **硬件加速组件：** 挂载 NVIDIA RTX 3060 设备，凭借 CUDA (Compute Unified Device Architecture) 技术框架，为上层特征提取及神经网络运算提供原生 GPU 并发浮点运算加速引擎。
    *   **核心关系型数据库：** 部署 MySQL 实例以提供基础的 ACID 支持并处理核心结构化元数据。
    *   **边界服务通信网关：**
        *   **数据采集驱动引擎：** 基于 DrissionPage Chromium CDP 底层协议获取外部平台复杂动态数据。
        *   **大语言模型云端总线：** 集成前沿大语言模型 DeepSeek API 网关，增强系统的 NLP 语义理解与推理上限。
        *   **外部流量平台：** 对接广域网中的 TikTok Platform 生产节点，构建多维数据流抓取的源数据蓄水池。
*   **核心功能映射：** 从微观和硬件维度支撑前置业务层的“视频特征提取的底层算力调度”、“基于 DrissionPage 的采集爬虫链路（Acquisition Engine）”及第三方安全隔离请求工作。

## X.4 系统内部跨层请求流转与 GPU 调度链路

为确保多模态系统运转的安全可控并高效压榨系统算力，请求在四层架构间的流转机制采用以下设计规范：

1.  **表现层触达与接收：** 终端用户在 **表现层** 的 Django Templates/SimpleUI 用户交互组件下发“TikTok 短视频多模态联合分析”处理指令。
2.  **业务路由拦截与流量分发：** 请求透传至 **业务逻辑层**（Django Framework Router）进行负载验证。为保护前端呈现主线程的高可用响应，控制器捕获任务信号后立即转交给 **异步任务调度器**（Django-Q/Celery）打入高可用执行堆栈中，返回非阻塞异步确认至表现层。
3.  **网关调用与数据源爬取：** 异步 Worker 节点在 **业务逻辑层** 拾取该高资源瓶颈任务，向下调用 **基础设施层** 的外部通信总线（DrissionPage Scraper 组件），建立无头进程深入 TikTok 平台模拟用户行为链，拉取结构化元数据流与原始 MP4/MP3 等二进制资产。
4.  **硬件下沉交互与 AI 分析推理：** 该环节为系统的核心负载：
    *   **特征提取分流：** 被回传的媒体资产在交由 **业务逻辑层** 节点负责调度执行期间，通过 Python 封装层级的驱动，向下深度击穿调用 **基础设施层** 的 NVIDIA RTX 3060 逻辑算力通道池。
    *   **GPU 加速赋能：** 基于 CUDA 驱动引擎的支持，OpenCV 及 Librosa 引擎执行基于张量层面的矩阵解算加速，使极度耗费浮点运算力的逐帧光流视频解析能够达到满载流片速度；在文本情感分析领域，则同步向 DeepSeek API 设施发起云端异步回调校验任务。
    *   **多模态融合预测：** 各端获取到的量化特征矩阵数据全部归口汇总至由 XGBoost 构建的核心预测业务组件，展开分类决策拟合。
5.  **ORM 序列化归档：** XGBoost 产出的深度分析评估分值与特征权重等推论结果打包成业务对象实体，借由 **数据访问层** 框架内聚的 ORM 的隐式执行引擎转换为主键与外键关联关系序列，最后落库下行至 **基础设施层** 存储基座中的 MySQL 进行高吞吐锁存。
6.  **结果逆向回调呈现：** 当底座持久化流程顺利回滚事务确认结束时，挂载在 **表现层** 侦听 WebSocket 长连接（或定时轮询接口）的回调服务激活通知包，指示 ECharts/Chart.js DOM 组件挂接到 REST 接口并请求 **数据访问层** 最近调取的元数据，从而将枯燥的数据降维投射为可交互的直观视图呈现给终端业务用户，至此彻底闭环整个逻辑架构请求链。
